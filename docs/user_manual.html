Quick Reference

// feynpy user manual

A Python library for defining quantum field theory models and extracting Feynman rules symbolically, built on top of Symbolica and Spenso.

Core imports

from symbolica import S, Expression
from symbolic.vertex_engine import I
from symbolic.spenso_structures import gauge_generator, structure_constant, weak_gauge_generator
from model import (
    Field, GaugeGroup, GaugeRepresentation, IndexType, Model,
    DeclaredLagrangian, CompiledLagrangian,
    CovD, PartialD, Gamma, Gamma5, FieldStrength,
    GaugeFixing, GhostLagrangian, GhostField,
    # pre-built index constants
    SPINOR_INDEX, LORENTZ_INDEX, COLOR_FUND_INDEX, COLOR_ADJ_INDEX,
    WEAK_FUND_INDEX, WEAK_ADJ_INDEX,
    # flavor
    flavor_index, Parameter,
)

— Workflow at a glance

1
Declare index types Use pre-built constants or create custom IndexType objects.
2
Declare fields Instantiate Field with spin, symbol, and which index types it carries.
3
Declare gauge groups & representations Create GaugeRepresentation and GaugeGroup objects.
4
Declare the model with Model(...) Use CovD, FieldStrength, GaugeFixing, spenso tensors, or plain field products.
5
Compile with model.lagrangian() Returns a compiled CompiledLagrangian object.
6
Extract rules with model.feynman_rule(...) Pass explicit fields to get one vertex, or pass nothing to discover all. 
# Complete minimal example — scalar phi^4
Phi = Field("Phi", spin=0, self_conjugate=True, symbol=S("phi"))
lam  = S("lam")

model     = Model(lam * Phi * Phi * Phi * Phi)
lagrangian = model.lagrangian()
vertex    = lagrangian.feynman_rule(Phi, Phi, Phi, Phi)

01 Index Types

Index types pair a Spenso Representation with a string kind used to match indices through the compilation pipeline. The library ships six pre-built constants — use them directly.

SPINOR_INDEX
bis(4) · kind="spinor"
LORENTZ_INDEX
mink(4) · kind="lorentz"
COLOR_FUND_INDEX
cof(3) · kind="color_fund"
COLOR_ADJ_INDEX
coad(8) · kind="color_adj"
WEAK_FUND_INDEX
cof(2) · kind="weak_fund"
WEAK_ADJ_INDEX
coad(3) · kind="weak_adj"

Custom index type

from symbolica.community.spenso import Representation
from model import IndexType

# IndexType(name, representation, kind, prefix=...)
MY_FUND = IndexType("MyFund", Representation.cof(3), "my_fund", prefix="m")

Flavor index

# flavor_index(name, dimension, prefix=...)  — sets is_flavor=True automatically

02 Fields

A Field declares one particle species. It specifies the spin, the Symbolica symbol(s), which index types it carries, and optional quantum numbers.

ParameterDescription
nameHuman-readable identifier string.
spin0, Fraction(1,2), or 1.
self_conjugateTrue for real scalars and gauge bosons; False for complex fields and fermions.
symbolSymbolica atom for this field.
conjugate_symbolRecommended when self_conjugate=False; used as the exported conjugate species.
indicesTuple of IndexType objects the field carries.
quantum_numbersDict of charge labels, e.g. {"Q": S("e")}. Required for abelian gauge coupling.
kindSet to "ghost" for ghost fields.
flavor_indexThe IndexType to use for flavor expansion.
class_membersTuple of member name strings, one per flavor generation.

Field type examples

Real scalar
spin=0, self_conjugate=True, no indices
Complex scalar
spin=0, self_conjugate=False, symbol + conjugate_symbol
Dirac fermion
spin=½, self_conjugate=False, carries SPINOR_INDEX
Gauge boson
spin=1, self_conjugate=True, carries LORENTZ_INDEX (+ adj for non-abelian)
Ghost
spin=0, kind="ghost", self_conjugate=False, carries adj index
from fractions import Fraction

# Real scalar
Phi  = Field("Phi", spin=0, self_conjugate=True, symbol=S("phi"))

# Complex scalar with U(1) charge
PhiC = Field("PhiC", spin=0, self_conjugate=False,
             symbol=S("phiC"), conjugate_symbol=S("phiCdag"),
             quantum_numbers={"Q": S("Qphi")})

# Dirac fermion with spinor index and U(1) charge
Psi  = Field("Psi", spin=Fraction(1,2), self_conjugate=False,
             symbol=S("psi"), conjugate_symbol=S("psibar"),
             indices=(SPINOR_INDEX,),
             quantum_numbers={"Q": S("Qpsi")})

# Quark — spinor + color
Quark = Field("q", spin=Fraction(1,2), self_conjugate=False,
              symbol=S("q"), conjugate_symbol=S("qbar"),
              indices=(SPINOR_INDEX, COLOR_FUND_INDEX))

# Photon (abelian gauge boson)
Photon = Field("A", spin=1, self_conjugate=True,
               symbol=S("A"), indices=(LORENTZ_INDEX,))

# Gluon (non-abelian gauge boson)
Gluon  = Field("G", spin=1, self_conjugate=True,
               symbol=S("G"), indices=(LORENTZ_INDEX, COLOR_ADJ_INDEX))

# Ghost (use GhostField for the ghost_of shortcut)
from model import GhostField
GhostG = GhostField("ghG", ghost_of=Gluon, self_conjugate=False,
                    symbol=S("ghG"), conjugate_symbol=S("ghGbar"),
                    indices=(COLOR_ADJ_INDEX,))
Conjugated fields: use Field.bar in Lagrangian expressions and feynman_rule calls, e.g. Psi.bar, Higgs.bar.

03 Gauge Groups & Representations

For non-abelian groups, first create a GaugeRepresentation for each matter representation, then create the GaugeGroup.

GaugeRepresentation

ParameterDescription
indexThe IndexType for the representation (e.g. COLOR_FUND_INDEX).
generator_builderFunction that builds the generator tensor: gauge_generator for the built-in color/SU(3) fundamental, weak_gauge_generator for the built-in SU(2) doublet, or a custom builder for other groups.
nameLabel string.
slotFor fields with repeated same-kind indices, specify which slot (0-based) is the active one.
slot_policy"sum" to sum over all matching slots instead of requiring uniqueness.

GaugeGroup

ParameterDescription
nameIdentifier string.
abelianTrue for U(1) groups.
couplingSymbolica symbol for the coupling constant.
gauge_bosonThe gauge boson Field.
chargeFor abelian groups: the quantum number key to read from matter fields (e.g. "Q").
structure_constantFor non-abelian groups: the structure-constant builder (e.g. structure_constant).
representationsTuple of GaugeRepresentation objects for matter fields.
ghost_fieldThe ghost Field — required for non-abelian GhostLagrangian(...).
# Representation for color fundamental
COLOR_FUND_REP = GaugeRepresentation(
    index=COLOR_FUND_INDEX, generator_builder=gauge_generator, name="fundamental"
)

# U(1) QED
U1QED = GaugeGroup(name="U1QED", abelian=True, coupling=S("e"),
                   gauge_boson=Photon, charge="Q")

# SU(3) QCD
SU3C = GaugeGroup(name="SU3C", abelian=False, coupling=S("gs"),
                  gauge_boson=Gluon, ghost_field=GhostG,
                  structure_constant=structure_constant,
                  representations=(COLOR_FUND_REP,))

# SU(2) weak
WEAK_DOUBLET_REP = GaugeRepresentation(
    index=WEAK_FUND_INDEX, generator_builder=weak_gauge_generator, name="doublet"
)
SU2L = GaugeGroup(name="SU2L", abelian=False, coupling=S("g2"),
                  gauge_boson=W, ghost_field=GhostW,
                  structure_constant=weak_structure_constant,
                  representations=(WEAK_DOUBLET_REP,))

04 Flavor Indices & Parameters

Flavor indices let you write compact multi-generation expressions and later expand them into per-member rules.

# 1. Create a flavor index
Generation = flavor_index("Generation", 3, prefix="f")

# 2. Field with flavor index + class members
lepton = Field("l", spin=Fraction(1,2), self_conjugate=False,
               indices=(Generation, SPINOR_INDEX),
               symbol=S("l"), conjugate_symbol=S("lbar"),
               flavor_index=Generation,
               class_members=("e", "mu", "ta"))
e, mu, ta = lepton.class_members

# 3. Flavor-aware parameter
#    indices= lists the index types the parameter carries
#    components= maps (i,j)->value for sparsity (0 drops that component)
Yu = Parameter("Yu", indices=(Generation, Generation))

# Diagonal-only parameter
Ydiag = Parameter("Ydiag", indices=(Generation, Generation),
                  components={(i,j): 0 for i in range(1,4) for j in range(1,4) if i!=j})
Usage in expressions: call the parameter like a function: Yu(f, h). Summed flavor indices in feynman_rule(..., flavor_expand=True) automatically produce one rule per generation combination.

05 Declare a Model

Pass the source declaration directly into Model(...), or explicitly via Model(..., lagrangian_decl=...). Multiple terms are summed with +. There is no separate source-level Lagrangian(...) constructor in the current API.

Gauge metadata: declarations involving CovD need the relevant gauge groups present in the model metadata. For FieldStrength(...), GaugeFixing(...), and GhostLagrangian(...), the group object is carried directly by the declaration. 
# Simple local model — no gauge_groups needed
m = Model(S("lam") * Phi * Phi * Phi * Phi)

# Gauge-aware model — include the relevant gauge-group metadata
mu = S("mu")
m = Model(
    I * Psi.bar * Gamma(mu) * CovD(Psi, mu),
    gauge_groups=(U1QED,),
)

# Multi-term model
m = Model(
    I * Quark.bar * Gamma(mu) * CovD(Quark, mu)
    - (Expression.num(1)/Expression.num(4))
      * FieldStrength(SU3C, mu, S("nu"), S("a"))
      * FieldStrength(SU3C, mu, S("nu"), S("a"))
    + GaugeFixing(SU3C, xi=S("xi"))
    + GhostLagrangian(SU3C),
    gauge_groups=(SU3C,),
)

# Compile
L = m.lagrangian()

Model validation

# Run the source-level validation report before extracting rules
report = m.validate()
print(report.ok)       # True if no issues
print(report.issues)  # list of Issue(code, message)

06 Operator Catalog

Every builder below is a valid source-level building block inside Model(...) or Model(..., lagrangian_decl=...).

Local monomials

Field * Field * ...                        # plain product
Psi.bar * Psi * Phi                         # Yukawa
Psi.bar * Gamma(mu) * Psi * Photon          # vector current
Psi.bar * Gamma(mu) * Gamma5() * Psi * V   # axial current

Derivative operators — PartialD

PartialD(Phi, mu)                          # ∂_μ Φ
PartialD(PartialD(Phi, mu), nu)            # ∂_μ ∂_ν Φ (nested)
Psi.bar * Gamma(mu) * PartialD(Psi, nu) * Phi  # fermion derivative operator

Gauge covariant derivative — CovD

mu = S("mu")
# Fermion kinetic: I * ψ̄ γ^μ D_μ ψ
I * Psi.bar * Gamma(mu) * CovD(Psi, mu)

# Complex scalar kinetic: (D_μ Φ†)(D^μ Φ)
CovD(PhiC.bar, mu) * CovD(PhiC, mu)

# Convention: D_μ = ∂_μ - igA_μ

Field strength — FieldStrength

# Abelian (no adjoint index)
FieldStrength(U1QED, mu, nu)

# Non-abelian (adjoint index required)
FieldStrength(SU3C, mu, nu, S("a"))

# Yang-Mills kinetic term
-(Expression.num(1)/Expression.num(4)) * FieldStrength(SU3C, mu, nu, S("a")) * FieldStrength(SU3C, mu, nu, S("a"))
Strict rules: non-abelian FieldStrength requires an explicit adjoint index; abelian must not have one. All adjoint indices must be contracted (no open color).

Gauge fixing & ghosts

GaugeFixing(SU3C, xi=S("xi"))   # linear covariant gauge
GhostLagrangian(SU3C)          # ordinary non-abelian Faddeev-Popov ghosts (group needs ghost_field)

07 Extracting Feynman Rules

Call feynman_rule on the Model object or on a compiled CompiledLagrangian.

Targeted extraction

L = Model().lagrangian()

# Pass fields explicitly to get one vertex
vertex = L.feynman_rule(Psi.bar, Psi)

# Custom external momenta (default: q1, q2, q3, ...)
vertex = L.feynman_rule(Psi.bar, Psi, Photon,
                        momenta=[S("p_in"), S("p_out"), S("k")])

Key options

OptionDefaultDescription
simplifyTrueRun symbolic simplification on the result.
simplify_gammaFalseApply Dirac algebra simplification to gamma chains.
include_deltaFalseInclude the overall momentum-conservation Delta(...) factor.
strip_externalsTrueStrip external wavefunction factors (recommended default).
flavor_expandFalseTrue, one index, or a tuple of indices to expand over generations.
key_format"names"For zero-argument extraction: dict keys are "names" (string tuples) or "fields" (Field objects).

Output conventions

Reading the output: Delta(...) = momentum conservation factor · gamma(...) = open Dirac structure · t(...), f(...) = generator / structure constant · g(...) = metric · pcomp(p, mu) = momentum component.

08 Vertex Discovery & Filtering

Call feynman_rule() with no arguments to enumerate every available vertex. Use structured filtering methods to inspect without computing all expressions.

# Enumerate all vertices — returns dict keyed by name tuples
all_rules = L.feynman_rule(simplify=False)
for sig, expr in all_rules.items():
    print(sig, "→", expr)

# Use key_format="fields" for Field-object keys
all_rules = L.feynman_rule(simplify=False, key_format="fields")

vertex_signatures — lightweight inspection

# All signatures
for sig in L.vertex_signatures():
    print(sig.names, "arity=", sig.arity, "terms=", sig.term_count)

# Filter by arity
three_point = L.vertex_signatures(arity=3)

# Exact signature match (order-independent)
exact = L.vertex_signatures(signature=(Gluon, Quark.bar, Quark))

# Contains specific fields
ghost_related = L.vertex_signatures(contains_fields=(GhostG.bar, GhostG))

vertex_report

report = L.vertex_report(arity=3)
print(report.matched_signatures, "of", report.total_signatures)
for sig in report.signatures:
    print(sig.names)
Tip: use vertex_signatures / vertex_report to inspect a large model before calling feynman_rule, which actually computes the expressions.

09 Flavor Expansion

Use flavor_expand=True to expand all flavor/generation indices. Alternatively, pass one index or a tuple of indices to expand only selected flavor indices, turning generic multi-generation vertices into explicit per-member vertices. 

# Compact (generation-generic) rules
rules_compact  = L.feynman_rule(simplify=True)

# Fully expanded rules (all flavor indices)
rules_expanded = L.feynman_rule(simplify=True, flavor_expand=True)

# Expand only one specific index
rules_gen      = L.feynman_rule(simplify=True, flavor_expand=Generation)

# Specific member vertex (requires flavor_expand)
vertex_e_mu    = L.feynman_rule(e.bar, mu, Phi, flavor_expand=True)

# Discovery also supports flavor_expand
sigs = L.vertex_signatures(flavor_expand=True)
Off-diagonal zeroes: if a parameter has components={(i,j): 0} for i≠j, off-diagonal member combinations are automatically absent from the expanded rules.

10 Symbolica Export

Call to_symbolica() on a compiled CompiledLagrangian to get a flat Expression for display, collection, differentiation, and other commutative Symbolica manipulations.

Default export

expr = L.to_symbolica()
print(expr)

# Standard Symbolica operations work directly
expr.expand()
expr.derivative(S("Phi"))
expr.coefficient(S("lam"))

Coordinate-dependent export

# derivative_style="coordinate" makes spacetime dependence explicit
expr_coord = L.to_symbolica(derivative_style="coordinate")
expr_coord.derivative(S("x_mu"))

Flavor-expanded export

# Compact (generation-generic)
compact  = L_fl.to_symbolica()

# Expanded into per-member expressions
expanded = L_fl.to_symbolica(flavor_expand=True)
Ordering caveat: to_symbolica() is a commutative export view. Fermion and ghost ordering is not preserved there, so ordering-sensitive manipulations should stay on the compiled terms / operator layer.

11 Applying Operators

The operator layer acts on the compiled CompiledLagrangian term-by-term, following a graded Leibniz rule. It is the foundation for field redefinitions, substitutions, gauge variations, and BRST transformations.

Required imports

from lagrangian.operator_action import (
    FieldOperator, TermOperator,
    OperatorAtomResult, OperatorSummand,
    OperatorExpansionError,
    replacement_operator, single_field_result,
    partial,
)

Simple field replacement

# replacement_operator — the simplest case: Phi → Chi everywhere
op = replacement_operator("Phi_to_Chi", {Phi: Chi()})
L_new = L.apply_operator(op)

# Also available on Model as a convenience forwarder: it compiles first, then applies
L_new = model.apply_operator(op)

FieldOperator — slot-wise action

Acts once per matched field slot, expanding via the graded Leibniz rule. The callback receives an FieldOccurrence and returns None (skip) or an OperatorAtomResult.

# Phi → a·Chi + b·Omega  (one summand per slot)
op = FieldOperator(
    "phi_to_combo",
    on_field=lambda occ: (
        None if occ.field is not Phi else
        OperatorAtomResult(summands=(
            OperatorSummand(coefficient=S("a"), replacement=(Chi(),)),
            OperatorSummand(coefficient=S("b"), replacement=(Omega(),)),
        ))
    ),
)

# Odd operator (parity=1 triggers graded Leibniz signs)
s = FieldOperator("s", parity=1, on_field=lambda occ: ...)

TermOperator — whole-term action

# Acts on each InteractionTerm as a whole (no Leibniz expansion)
from dataclasses import replace
scale2 = TermOperator(
    "scale2",
    apply_to_term=lambda term: (replace(term, coupling=2 * term.coupling),),
)

Spacetime derivative operator — partial(mu)

# Applies ∂_μ to every slot (Leibniz expansion)
L_partial = L.apply_operator(partial(mu))

# Restrict to a specific field
L_partial = L.apply_operator(partial(mu, on=Phi))

# Declarative shortcut inside lagrangian_decl=... (same as PartialD(Phi, mu))
dPhi = partial(mu, Phi)

Composition and algebra

# Apply a sequence of operators left-to-right
L_out = L.apply_operators(op1, op2, op3)

# Graded commutator [A, B] = A∘B − (−1)^{|A||B|} B∘A
L.operator_bracket(opA, opB).to_symbolica()

# Expansion guard: raises OperatorExpansionError if too many terms generated
try:
    L.apply_operator(op, max_generated_terms=10)
except OperatorExpansionError as exc:
    print(exc)
Derivative fan-out: when a replacement operator maps one field to a product, any pre-existing derivative on that slot fans out via Leibniz automatically — no manual handling needed.

12 IBP Normal Form

Two scalar Lagrangians that differ by a total derivative are physically equivalent. ibp_normal_form() computes the current v1 scalar IBP normal form of a compiled Lagrangian, so matching normal forms certify equality up to boundary terms within that supported domain.

# The identity: φ²□φ ≃ −2φ(∂_μφ)(∂_μφ)  (mod total derivative)
L1_expr = c * phi * phi * PartialD(PartialD(phi, mu), mu)
L2_expr = -2 * c * phi * PartialD(phi, mu) * PartialD(phi, mu)

L1 = Model(L1_expr).lagrangian()
L2 = Model(L2_expr).lagrangian()

NF1 = L1.ibp_normal_form()
NF2 = L2.ibp_normal_form()
assert NF1.terms == NF2.terms

# Check equivalence: their difference vanishes in normal form
delta_NF = Model(L1_expr - L2_expr).lagrangian().ibp_normal_form()
assert delta_NF.terms == ()  # empty ⟹ equal modulo boundary terms
Current scope: this v1 normalizer is for scalar compiled terms with ordinary partial derivatives. Unsupported gauge / spinor / ordering-sensitive structures are rejected instead of being normalized heuristically.
Tip: use ibp_normal_form() to verify scalar operator identities or eliminate scalar higher-derivative redundancies before extracting Feynman rules.

13 Gauge Variation

gauge_variation(group=..., parameter=...) returns a FieldOperator implementing the infinitesimal gauge variation for a single group. It follows the graded Leibniz rule and materialises the gauge parameter α as a synthetic scalar field carrying the group's adjoint index.

Transformation rules

FieldVariation δ
Matter rep R: Φ_i+ig α^a T^a_{ij} Φ_j
Matter rep R: Φ̄_i−ig α^a Φ̄_j T^a_{ji}
U(1) charge q: Φ+igq α Φ
Abelian gauge boson A_μ+∂_μ α
Non-abelian gauge boson A^a_μ+∂_μ α^a − g f^{abc} α^b A^c_μ
from lagrangian.operator_action import gauge_variation
from symbolic.tensor_canonicalization import canonize_full

# Compute δL under a gauge group
delta = gauge_variation(group=U1QED, parameter="alpha")
dL    = L.apply_operator(delta)

# Verify invariance using canonize_full
canon = canonize_full(
    dL.to_symbolica().expand(),
    lorentz_indices=(mu,),
)
assert canon.to_canonical_string() == "0"  # ⟹ gauge invariant

Wrong-charge sanity check

# Gauge variation of a charge-violating term is non-zero
L_wrong = Model(S("y") * eta_neutral.bar() * eta_neutral() * Phi_charged).lagrangian()
dL_wrong = L_wrong.apply_operator(gauge_variation(group=U1QED, parameter="alpha"))
# dL_wrong.to_symbolica().expand() ≠ 0  — charge not conserved
Non-abelian two-part variation: the gauge-boson variation has an inhomogeneous piece (+∂_μ α^a) and a homogeneous piece (−g f^{abc} α^b A^c_μ). Both are produced automatically by gauge_variation.

14 BRST Transformations

brst_transformation(group, ghost, antighost=None, auxiliary=...) returns an odd FieldOperator that implements the full BRST transformation for one gauge group. The ghost must be declared with kind="ghost" and ghost_of=<gauge_boson>.

Sign conventions

FieldBRST rule s(·)
Gauge boson A^a_μD_μ c^a = ∂_μ c^a + g f^{abc} A^b_μ c^c
Ghost c^a−(g/2) f^{abc} c^b c^c
Antighost c̄^aB^a (auxiliary field)
Auxiliary B^a0
Matter Φ_i (non-abelian)+ig c^a T^a_{ij} Φ_j
Matter Φ̄_i (non-abelian)−ig c^a Φ̄_j T^a_{ji}
Matter Φ (abelian, charge q)+igq c Φ
from lagrangian.operator_action import brst_transformation

# Declare ghost and auxiliary fields
c    = Field("c", spin=0, kind="ghost", ghost_of=Gluon,
             self_conjugate=False, conjugate_symbol=S("cbar"),
             indices=(COLOR_ADJ_INDEX,),
             quantum_numbers={"GhostNumber": 1})
Baux = Field("B", spin=0, self_conjugate=True, indices=(COLOR_ADJ_INDEX,))

s = brst_transformation(group=SU3C, ghost=c, auxiliary=Baux)

# Act on individual fields
mu, a = S("mu"), S("a")
print(Gluon(mu, a).apply_operator(s).to_symbolica().expand())  # s A^a_μ
print(c(a).apply_operator(s).to_symbolica().expand())            # s c^a

Nilpotency check s² = 0

# Verify s²=0 for the gauge boson
s2_G = Gluon(mu, a).apply_operator(s).apply_operator(s).to_symbolica().expand()
canon = canonize_full(s2_G, run_gamma=False, run_color=False, infer_indices=True)
assert canon.to_canonical_string() == "0"

BRST-exact gauge fixing

# Build s(K) to get the gauge-fixed Lagrangian
K = Model(
    gauge_groups=(SU3C,),
    lagrangian_decl=(
        c.bar(a) * PartialD(Gluon(mu, a), mu)
        + xi / Expression.num(2) * c.bar(a) * Baux(a)
    ),
).lagrangian()
L_gf = L_YM + K.apply_operator(s)  # full gauge-fixed L
Ghost validation: brst_transformation validates that the ghost field is declared with kind="ghost" and that ghost_of matches the group's gauge boson. A mismatch raises ValueError at construction time.

15 Expression Manipulation

Once a compiled Lagrangian is exported via to_symbolica(), standard Symbolica methods (expand, collect, coefficient) work directly on that commutative view. The main subtlety is that wildcard labels (e.g. mu_) are not interpreted by the native coefficient(...), so use the helper or a manual match loop instead.

Standard operations

expr = L.to_symbolica()

expr.expand()
expr.coefficient(S("xi"))     # works for literal symbols
expr.coefficient(S("gS"))     # extract coupling-constant dependence

Wildcard-aware coefficient extraction

from lagrangian.symbolica_export import pattern_coefficient

G  = Gluon.symbol
mu1_, a1_, mu2_, a2_ = S("mu1_", "a1_", "mu2_", "a2_")  # trailing _ = wildcard
pattern = G(mu1_, a1_) * G(mu2_, a2_)

# Helper handles wildcard matching automatically
print(pattern_coefficient(expr, pattern))

Pure Symbolica workaround (no helper)

seen = {}
for match in expr.match(pattern, min_level=0, max_level=0, partial=True):
    factor = pattern.replace_wildcards(match)
    seen.setdefault(factor.to_canonical_string(), factor)

total = Expression.num(0)
for factor in seen.values():
    total += expr.coefficient(factor)
Wildcard convention: Symbolica treats symbols whose names end with _ as wildcards in match(...) and replace_wildcards(...) calls. Use this for index-agnostic pattern extraction.
