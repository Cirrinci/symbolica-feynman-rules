\chapter{Implementation: From \texttt{Model} to Feynman Rule}
\label{ch:internals}
% ==============================================================

The previous chapter described \textit{how to use} the toolkit. This chapter opens the box: it follows one Lagrangian all the way from the declarative Python object the user writes down to the Symbolica expression that comes out as a Feynman rule, and explains what each intermediate layer is responsible for. Nothing in this pipeline is a new physical ingredient --- the physics is entirely fixed already, by the conventions of Section~\ref{sec:feynpy-gauge} and by the three-step canonical-quantization algorithm of Section~\ref{sec:feynrules-algorithm}. What the implementation adds is bookkeeping: it has to track, mechanically and without error, \emph{which} index on \emph{which} field slot is which, across covariant-derivative expansions, permutation sums and tensor simplifications, for fields carrying any combination of Lorentz, Dirac, colour, weak-isospin and flavour indices.

It is useful to keep one concrete picture in mind throughout the chapter: a term such as
\begin{equation}
    i\,\bar q\,\gamma^\mu D_\mu q \;\supset\; g_s\,\bar q_{i}\,\gamma^\mu\,t^a_{ij}\,q_{j}\,G^a_\mu
\end{equation}
is, by the end of the pipeline, turned into the QCD quark--gluon vertex $-i g_s\,\gamma^\mu\,t^a_{ij}$. Figure~\ref{fig:pipeline-overview} shows the stages that take us there, and the rest of this chapter walks through each of them, layer by layer, ending with the engine that performs the actual contraction.

In short:
\begin{enumerate}
    \item \textbf{Declaration} — the user writes \py{Model(...)}: fields, gauge groups and a symbolic Lagrangian built from \py{CovD}, \py{PartialD}, \py{FieldStrength}, \py{Gamma}, etc. This is purely declarative; no physics has been expanded yet.
    \item \textbf{Lowering / compilation} — \py{model.lagrangian()} expands every declarative factor (covariant derivatives, field strengths, gauge fixing, ghosts) into elementary monomials of bare fields, and packages each monomial as an \py{InteractionTerm}: a coupling, a tuple of field occurrences, and the derivatives acting on them. The result is a \py{CompiledLagrangian}.
    \item \textbf{Flavor expansion (optional)} — generation/flavour indices are resolved into concrete per-member terms, dropping any vanishing components.
    \item \textbf{Contraction} — for a chosen set of external fields, \py{feynman_rule(...)} builds one \py{ExternalLeg} per field, normalizes everything to flat data via \py{to_vertex_kwargs(...)}, and calls the engine, \py{contract_to_full_expression}: the code-level realization of the three-step algorithm above.
    \item \textbf{Post-processing} — the raw contraction is multiplied by the universal overall $i$, optionally decorated with the momentum-conservation $\delta$, and simplified: metrics/deltas contracted, gamma chains reduced, colour factors simplified, using Spenso/idenso.
\end{enumerate}

Sections~\ref{sec:internals-symbolica-spenso}--\ref{sec:internals-postprocessing} go through these stages in more detail. A version of Figure~\ref{fig:pipeline-overview} suitable for browsing outside the thesis (e.g.\ rendered live on GitHub) is kept as a Mermaid source diagram in \texttt{viz/mermaid/thesis\_pipeline\_overview.md}.\footnote{Mermaid flowcharts are plain-text diagram sources; they render automatically wherever Markdown is viewed (GitHub, VS Code, \dots) but are not natively understood by \LaTeX{}, so the thesis figure above is the same diagram redrawn in \texttt{TikZ}.}

% --------------------------------------------------------------
\section{Symbolica and Spenso}
\label{sec:internals-symbolica-spenso}
% --------------------------------------------------------------

The whole toolkit is built on top of two libraries, and it is worth being precise about what each one is responsible for, since the split between them is exactly what makes the rest of this chapter possible.

\subsection*{Symbolica: the computer-algebra core}

\textsc{Symbolica}~\cite{symbolica} is a general-purpose computer-algebra system: a Rust engine, exposed through a Python package (\py{pip install symbolica}), for building and manipulating symbolic expressions. Every object the toolkit works with --- a coupling, a momentum, a gamma matrix, an entire Feynman rule --- is, in the end, one Symbolica \py{Expression}. A few properties of the library are directly responsible for design choices made throughout the toolkit:

\begin{itemize}
    \item \textbf{Native embedding, not a subprocess.} Unlike driving an external CAS (FORM, Mathematica) through files or pipes, Symbolica is linked directly into the Python process. There is no string round-tripping: \py{Expression} objects are first-class Python values that can be stored on a dataclass (as \py{InteractionTerm.coupling} is), compared, hashed, and passed around like any other object.
    \item \textbf{Speed, driven by polynomial algebra.} Symbolica's own benchmarks report multivariate GCD computations completed in seconds where Mathematica and SymPy take from a minute to over an hour~\cite{symbolica}; this is because fast multivariate GCD, factorization and interpolation over the rationals sit at the core of the engine, underneath the higher-level expression layer. This matters here because \py{contract_to_full_expression} (Section~\ref{sec:internals-contraction}) generates one term per permutation and relies on Symbolica's \py{.expand()}/\py{.collect()} to bring the resulting, often large, sums back to a compact polynomial-in-momenta form.
    \item \textbf{Pattern matching with wildcards.} Symbolica symbols whose name ends in \py{_} act as wildcards in \py{.match(...)}/\py{.replace(...)} calls. The toolkit uses this pervasively: index remapping inside the contraction engine is literally \py{coupling.replace(open_label, target_label)}, and user-facing helpers such as \py{pattern_coefficient} (Section~\ref{sec:feynpy-expr-manip}) are thin wrappers around the same mechanism.
    \item \textbf{One algebra for everything downstream.} Because the final object is a plain \py{Expression}, any Symbolica method --- \py{.expand()}, \py{.derivative()}, \py{.coefficient()}, series expansion, and so on --- keeps working after the Feynman rule has been extracted, with no extra glue. This is what lets \py{CompiledLagrangian} forward unknown attribute lookups straight to its Symbolica export (Section~\ref{sec:feynpy-expr-manip}).
\end{itemize}

\subsection*{Spenso: typed tensor indices}

Symbolica by itself has no notion of a tensor index: a symbol \py{G(mu,a)} is just a function applied to two opaque arguments, and nothing tells Symbolica that \py{mu} lives in a 4-dimensional Minkowski space, or that repeating the label \py{a} on two factors means ``sum over the adjoint representation of the gauge group''. \textsc{Spenso}~\cite{spenso} is a companion tensor-network library, built on top of Symbolica, that adds precisely this missing layer: every index becomes a \py{Slot} --- a pair \py{(Representation, abstract_index)} --- and tensors built from typed slots automatically know which of their indices are allowed to contract with which.

A \py{Representation} fixes the dimension \emph{and} the transformation law of an index. The toolkit only ever uses the following six constructors (Table~\ref{tab:spenso-representations}), each instantiated once at dimension-fixing time and then reused for every slot of that kind:

\begin{table}[htbp]
\centering
\begin{tabular}{@{}llp{6.7cm}@{}}

\texttt{Representation.bis(4)}   & yes & Dirac bispinor indices carried by fermion fields \\
\texttt{Representation.cof(3)}   & no  & $SU(3)$ colour fundamental index of a quark \\
\texttt{Representation.coad(8)}  & yes & $SU(3)$ colour adjoint index of a gluon/generator/structure constant \\
\texttt{Representation.cof(2)}   & no  & $SU(2)_L$ weak-isospin fundamental (doublet) index \\
\texttt{Representation.coad(3)}  & yes & $SU(2)_L$ weak-isospin adjoint (triplet) index \\
\bottomrule
\end{tabular}

\label{tab:spenso-representations}
\end{table}

The \emph{self-dual} column is not a technicality: it is the tensor-index counterpart of the representation theory of Section~\ref{ch:theory}. A colour \emph{fundamental} index (carried by a quark) and its dual, the \emph{antifundamental} index (carried by an antiquark), are genuinely different representations, so \py{cof(3)} is \emph{dualizable} --- Spenso keeps track of which of the two a given slot is, exactly as one must distinguish $q^i$ from $\bar q_i$. A Lorentz, Dirac or colour-\emph{adjoint} index, on the other hand, is equivalent to its own dual (there is a metric/Killing form to raise and lower it, Eq.~\ref{killing_prop}), so \py{mink}, \py{bis} and \py{coad} are self-dual and any two matching slots of that kind are allowed to contract directly.

Once representations are fixed, building a typed tensor is direct: a \py{TensorName} (a tagged tensor head recognised by the HEP tensor library, \py{TensorName.gamma()}, \py{TensorName.t()}, \py{TensorName.f()}, \dots) is called on a list of \py{Slot}s, and \py{.to_expression()} turns the result into an ordinary Symbolica \py{Expression} that can be multiplied into a Lagrangian term like any other factor:

\begin{minted}{python}
def gamma_matrix(left_spinor, right_spinor, lorentz):
    """gamma^mu, as a Spenso tensor: bispinor x bispinor x Minkowski."""
    return TensorName.gamma()(
        bispinor_index(left_spinor),   # Representation.bis(4)(left_spinor)
        bispinor_index(right_spinor),  # Representation.bis(4)(right_spinor)
        lorentz_index(lorentz),        # Representation.mink(4)(lorentz)
    ).to_expression()

def gauge_generator(adj_index, fund_left, fund_right):
    """t^a_{ij}: adjoint x fundamental x fundamental."""
    return TensorName.t()(
        color_adj_index(adj_index),    # Representation.coad(8)(adj_index)
        color_fund_index(fund_left),   # Representation.cof(3)(fund_left)
        color_fund_index(fund_right),  # Representation.cof(3)(fund_right)
    ).to_expression()
\end{minted}

\noindent (\texttt{symbolic.spenso\_structures}; \py{structure_constant} for $f^{abc}$ and the weak-sector analogues follow the identical pattern.) Because every slot already carries its \py{Representation}, Spenso --- not the toolkit --- is the one that knows two \py{a}-labelled adjoint slots on \py{t(a,i,j)} and \py{G(mu,a)} are contractible; the toolkit's own bookkeeping (Section~\ref{sec:internals-indices}) only needs to track \emph{labels}, never representations directly.

Finally, a small companion module, \textbf{idenso}, supplies physics-aware simplification passes over Spenso tensor networks: \py{simplify_metrics} contracts Kronecker-delta/metric chains, \py{simplify_color} applies $SU(N)$ colour identities (Fierz-type collapses of \py{t}/\py{f} networks), and \py{simplify_gamma} performs Clifford algebra on chains of $\gamma$ matrices. The toolkit composes all three into one pass, \py{simplify_invariants} (\texttt{symbolic.spenso\_structures}), run in a fixed order --- metrics, colour, gamma, metrics again, since each pass can expose new metric pairs left by the previous one --- and it is this composed pass, together with the tensor-canonicalization step of Section~\ref{sec:internals-postprocessing}, that turns the raw permutation sum from \py{contract_to_full_expression} into the compact vertex printed by \py{feynman_rule(...)}.

In short: \textbf{Symbolica manipulates expressions; Spenso tells Symbolica what the indices inside those expressions mean, and idenso knows the physics-specific identities those typed indices satisfy.} The toolkit's own \py{IndexType} (Section~\ref{sec:feynpy-indices}) is the bridge between the three: it pairs one Spenso \py{Representation} with a plain string \emph{kind}, used everywhere internally as a cheap key to decide which indices are allowed to contract with which without repeatedly inspecting the representation itself.

% --------------------------------------------------------------
\section{Indices as Typed Slots}
\label{sec:internals-indices}
% --------------------------------------------------------------

Every index that appears anywhere in the pipeline --- on a field, on a derivative, on a coupling --- is tagged with an \py{IndexType(name, representation, kind, prefix)}. The \emph{representation} is the Spenso object that fixes the dimension and transformation law (fundamental, adjoint, Minkowski, bispinor, \dots); the \emph{kind} is a plain string (\py{"lorentz"}, \py{"spinor"}, \py{"color_fund"}, \dots) used as a fast key throughout the compiler to match indices without repeatedly inspecting the representation. A given field, e.g.\ a quark, simply carries a tuple of \py{IndexType} objects, one per slot it owns.

An index becomes a concrete \emph{label} (a Symbolica symbol such as \py{mu} or an automatically generated dummy \py{mu_1}) only once a specific occurrence of a field is placed inside a term. This label bookkeeping is centralized in one small object:

\begin{minted}{python}
@dataclass(frozen=True)
class FieldOccurrence:
    field: Field
    conjugated: bool = False
    labels: Mapping = field(default_factory=dict)   # slot -> label
\end{minted}

Given a full \py{InteractionTerm} (a tuple of \py{FieldOccurrence}, plus any derivatives), the property \py{index_bindings} groups \emph{every} slot and derivative index in the term by \py{(IndexType, label)}. This single grouping is what tells the rest of the pipeline whether an index is:
\begin{itemize}
    \item \textbf{internal / contracted} — the same label occurs on exactly two slots (e.g.\ a repeated colour index summed inside the term), or
    \item \textbf{open / external} — the label occurs once, and must therefore be identified with an index carried by one of the external legs when a vertex is later extracted.
\end{itemize}
This is exactly the distinction that later drives the index-remapping step inside \py{contract_to_full_expression} (Section~\ref{sec:internals-contraction}).

% --------------------------------------------------------------
\section{From a Declared Model to \texttt{InteractionTerm}s}
\label{sec:internals-lowering}
% --------------------------------------------------------------

Writing, schematically,
\begin{minted}{python}
Model(I * Quark.bar * Gamma(mu) * CovD(Quark, mu), gauge_groups=(SU3C,))
\end{minted}
does not yet produce anything expandable: it only builds a lightweight declarative tree of \emph{factors} (\py{_FieldFactor}, \py{CovariantDerivativeFactor}, \py{FieldStrengthFactor}, \dots) which record \emph{what the user wrote}, not yet its expansion. All physics-dependent expansion happens once, when \py{model.lagrangian()} is called, in two cooperating layers:

\begin{itemize}
    \item \texttt{feynpy.lowering} — pure rewriting rules that do not need to know about a specific gauge group: turning a Dirac kinetic monomial into elementary field/derivative factors, expanding a \py{FieldStrengthFactor} into its $\partial A - \partial A + g f A A$ pieces, etc.
    \item \texttt{compiler.gauge} (and its helpers \texttt{covariant\_core}, \texttt{matter\_actions}, \texttt{spectators}) — the layer that actually knows the frozen conventions of the toolkit: $D_\mu = \partial_\mu - ig A_\mu$, the sign of the non-abelian field strength, how a gauge-fixing or Faddeev--Popov ghost declaration expands once a \py{GaugeGroup} is supplied.
\end{itemize}

Concretely, \py{CovD(Quark, mu)} is rewritten schematically as
\[
    \texttt{PartialD(Quark,\ mu)} \;-\; i g\, t^a_{ij}\;\texttt{Gluon(mu,a)}\;\texttt{Quark(j)},
\]
each non-abelian \py{FieldStrength} expands into 2-, 3- and 4-point pieces, and every resulting elementary monomial is packaged into one \py{InteractionTerm}:

\begin{minted}{python}
@dataclass(frozen=True)
class InteractionTerm:
    coupling: object                       # Symbolica Expression
    fields: tuple[FieldOccurrence, ...]     # bare field factors, in order
    derivatives: tuple[DerivativeAction, ...] = ()
    dirac_bilinears: tuple[DiracBilinear, ...] = ()
    label: str = ""
    sector: str = ""
\end{minted}

A \py{CompiledLagrangian} is nothing more than a tuple of such \py{InteractionTerm}s. Note what has \emph{not} happened yet: no creation operators, no permutations, no external momenta --- \py{InteractionTerm} is a purely static, index-labelled description of one monomial of the Lagrangian, i.e. the right-hand side of Eq.~\ref{eq:general-term}. It is the direct code counterpart of $g_{\alpha_1\cdots\alpha_n}\,\partial\cdots\partial\,\phi^{\alpha_1}\cdots\phi^{\alpha_n}$.

\begin{tcolorbox}[notebox, title=Note]
Every \py{InteractionTerm} also records the field \emph{statistics} (boson/fermion, derived from its field content) and, when relevant, which pairs of fermionic slots form a closed Dirac bilinear ($\bar\psi \cdots \psi$). This structural information --- rather than re-deriving it from index labels at contraction time --- is what lets the engine assign the correct fermionic sign (Section~\ref{sec:internals-contraction}).
\end{tcolorbox}

\subsection*{Flavor expansion}
If a field carries a flavour index (Section~\ref{sec:feynpy-flavor}), \py{InteractionTerm}s stay generation-generic by default: a single term such as $\lambda_{ij}\,\bar\ell_i \ell_j \, \phi$ implicitly sums over $i,j$. \py{flavor_expand=True} (module \texttt{feynpy.flavor}) instead enumerates every member combination, substitutes the corresponding \py{Parameter} component (e.g.\ $\lambda_{e\mu}$), and \emph{drops} the term entirely if that component was declared to be zero. This produces one fully explicit \py{InteractionTerm} per generation combination, still in the same data structure described above.

\section{The Contraction Engine: \texttt{contract\_to\_full\_expression}}
\label{sec:internals-contraction}
% --------------------------------------------------------------

This function, in \texttt{symbolic.vertex\_engine}, is the only place where the actual physics of Section~\ref{sec:feynrules-algorithm} happens; everything else in the toolkit exists to feed it the right, uniformly-shaped input. Before it is called, the method \py{to_vertex_kwargs(...)} of \py{InteractionTerm} strips away all the model-layer vocabulary (\py{Field}, \py{IndexType}, \dots) and reduces the term plus the requested external legs to plain parallel Python sequences: \py{alphas}/\py{betas} (species of each internal field slot / external leg), \py{ps} (external momenta), \py{field_roles}/\py{leg_roles} (is this slot a $\psi$, a $\bar\psi$, a boson\dots), index labels and index \emph{kinds} per slot, and the coupling itself.

Given these flat inputs, \py{contract_to_full_expression} implements the three-step algorithm literally:

\begin{enumerate}
    \item \textbf{Sum over all pairings.} It loops over every permutation of the $n$ internal field slots against the $n$ external legs (\py{itertools.permutations}) --- this is precisely ``attach one creation operator per field and contract to the left'': each permutation is one way of assigning an external leg to an internal field slot, and the sum over all of them is the full Wick contraction.
    \item \textbf{Reject incompatible pairings, keep the sign.} A permutation contributes only if each internal slot is paired with a leg of the same species and compatible role (\py{factor_leg_compatible}); for fermions, the term is weighted by the sign of the permutation restricted to the fermionic slots (or, when the term carries a structural \py{closed_dirac_bilinears} pattern, by the sign fixed by that $\bar\psi\cdots\psi$ pairing instead of the raw permutation parity) --- the anticommutator sign of the canonical-quantization algorithm.
    \item \textbf{Remap indices and strip factors.} For each surviving permutation, every \emph{open} index label on the coupling/derivatives (Section~\ref{sec:internals-indices}) is substituted by the label carried by the leg it was paired with (\py{.replace(...)}), and each derivative contributes a momentum factor $-ip_\mu$ (Fourier convention) for the momentum of the leg it now acts on. All the contributions are summed into one Symbolica expression.
\end{enumerate}

\begin{minted}{python}
def contract_to_full_expression(
    *, alphas, betas, ps, x,
    derivative_indices=(), derivative_targets=None,
    statistics="boson", field_roles=None, leg_roles=None,
    field_index_labels=None, leg_index_labels=None,
    coupling=None, closed_dirac_bilinears=None,
):
    ...
    for perm in permutations(range(n)):
        term = coupling_with_open_indices_remapped_along(perm)
        if not all(compatible(i, perm[i]) for i in range(n)):
            continue
        if statistics == "fermion":
            term *= fermion_sign(perm, ...)
        term *= momentum_factors_from(derivatives, perm)
        total += term
    return total
\end{minted}

The result, \py{total}, is the vacuum matrix element of Section~\ref{sec:feynrules-algorithm} \emph{before} dropping the external wavefunctions/exponential and before multiplying by the overall $i$: those two remaining bookkeeping steps, together with the optional momentum-conservation $\delta$, are applied by the thin wrapper \py{vertex_factor(...)}, which is what \py{feynman_rule(...)} actually calls.

\begin{tcolorbox}[infobox, title={Why this generalizes so well}]
Nothing in \py{contract_to_full_expression} is specific to Lorentz or colour indices: an index is just a \py{(kind, label)} pair, and ``open'' vs.\ ``internal'' is determined purely by how many times a label occurs (Section~\ref{sec:internals-indices}). This is why the same engine derives QED, QCD, electroweak and ghost vertices without a single index-kind-specific branch.
\end{tcolorbox}

% --------------------------------------------------------------
\section{Post-processing and Canonicalization}
\label{sec:internals-postprocessing}
% --------------------------------------------------------------

The raw output of \py{contract_to_full_expression} is correct but redundant: it is a sum over $n!$ permutation terms, most of which vanish or duplicate each other once tensor identities are used, and it still contains internal Spenso tensor heads with arbitrary dummy-index names. Two further passes clean this up.

\texttt{symbolic.vertex\_postprocessing} applies the model-aware simplifications: contracting species Kronecker deltas (\py{simplify_deltas}), collapsing repeated bispinor indices via the Spenso metric (\py{simplify_spinor_indices}), and, if requested, running full Dirac-algebra simplification on closed gamma chains (\py{simplify_gamma=True} in \py{feynman_rule(...)}).

\texttt{symbolic.tensor\_canonicalization} then puts the surviving expression into a canonical form. Symbolica's own \py{canonize_tensors} only understands plain symbols with an \py{is_symmetric}/\py{is_antisymmetric} flag, not Spenso tensor heads, so the module temporarily \emph{swaps} every Spenso tensor (metric, generator, structure constant, \dots) for a plain symbol carrying the right symmetry, canonicalizes, and swaps back --- giving canonical, deterministic dummy-index names without losing the typed Spenso representation used everywhere else. On top of this generic pass, \py{canonize_full} adds a few physics-specific reductions used repeatedly when checking identities (Section~\ref{sec:feynpy-gauge-variation}, \ref{sec:feynpy-brst}): commuting flat partial derivatives, reducing repeated structure constants via the Jacobi identity, and recognizing antisymmetric-in-Lorentz-indices terms that must vanish.

The Symbolica expression returned by \py{feynman_rule(...)} is the result of all of the above: lowering $\to$ contraction $\to$ vertex factor $\to$ simplification, with every intermediate object (\py{InteractionTerm}, \py{ExternalLeg}, the flat contraction inputs) fully inspectable, which is what makes the toolkit usable both as a Feynman-rule calculator and as a general symbolic Lagrangian-manipulation environment (Section~\ref{sec:feynpy-apply-operator} onward).
